# TLM Dashboard Brand Assets

Real assets are inlined and live. To refresh after dropping a new logo here, run:

    .\.venv\Scripts\python.exe clients\client_tlm\creatives\inject_logos.py

## Files
- `TLM__logo_white.avif` — source: white "The Little Marionette" wordmark + marionette, transparent
  bg (client-supplied).
- `TLM_logo_navy.png` — recoloured (`#26303B`) copy used on the light dashboard (generated by
  `inject_logos.py`; kept as a reference of the recoloured asset).
- `Screenshot 2026-06-14 154949.png` — thelittlemarionette.com, the brand-palette reference.
- `inject_logos.py` — idempotent: recolours the source and sets the `src=` of the dashboard topbar
  (`<img class="brandlogo">`, navy on light cream) and the login (`<img class="client">`, white on
  dark slate). Re-run any time; it rewrites the `src` in place.

## Palette (sampled from thelittlemarionette.com)
- Page cream `#ECE5D8`, white cards `#FFFFFF`, warm ink `#2B2926`
- Steel-blue accent `#4F7290` (Google series), terracotta `#BD6B4C` (Trade Desk),
  revenue green `#2BA36F`, ROAS amber `#D98C1F`
- Login keeps a dark slate ground (`#16212B`) so the white logo reads; same steel-blue accent.

These values live in `dash/dashboard.html`'s `:root` (plus the JS `C{}` / `typeColor{}` chart
palette) and in `dash/main.py`'s LOGIN_HTML — edit them there, not here.
